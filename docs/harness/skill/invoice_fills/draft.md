关于备货单的介绍：
    用户会发送一个 excel 文件，
    （命名类似 "4.26-SP260414001-新棱镜备货-美国（4.28）-2.xlsx"，格式大概为“日期-SP单号-其它信息-国家（又是一个日期）”，注意，重要的是中间的单号和国家，在 SKILL 中要求 AI 自己从文件名中提取这两个信息，如果拿不到就和用户说缺少关键信息）
    然后要求执行填写报关单流程。

    这个 xlsx 文件中需要的数据在第三个 sheet。
    这第三个 sheet 第一行一定是下面这样的
    ```xlsx
    日期	SKU	品名	规格型号	发货量	单价	供货商	采购订单号	采购总价	商品名称	售价	总价	单位
    ```
    具体需要脚本提取的列有这些：
    ```xlsx
    品名    规格型号    发货量  商品名称    售价    总价    单位
    ```
    不过具体提取几行还是取决于第一列有几行有效数据。

```xlsx

```
这两个填写备货单的`商品名称`，英文名称直接做成规则映射，比如：

```xlsx
单箱产品申报数量*	单位*
```
这两个分别对应备货单的`发货量`和`单位`

```xlsx
产品材质*	产品海关编码*
```
这两个分别的填写方式直接复制填写报关资料时填写`其他申报要素`和`海关HS编码`的逻辑代码
但是`产品材质`要加点料，也就是不上英文，格式是这样的`en/cn`，比如
`尼龙/nylon`，`纸质+塑料/paper+plastic`

```xlsx
产品申报单价*
```
这个的填写方式依赖于 `单箱产品申报数量*` 和 `产品材质*`，是这样计算的：表带 + 硅胶 0.35/个
表带 + 尼龙  0.5/个
表带 + 皮革 0.5/个
表带 + 金属（或贱金属）  1/个
表壳（不看材质）  0.35/个
包装盒（不看材质） 0.32/个
手表保护套（不看材质） 0.35/个
其它 如果以上材质都不符合，直接填入`没有该材质的计算价格方式`


```xlsx

```
这两个填写很简单，填`无`即可

接下来来个硬家伙。

备货单里的 sku 都是已经归汇总好的库存 sku，没有具体告诉我们分几箱。
汇总是什么意思呢？就比如说有一个 msku 对吧，这个 msku 会对应一个本地 sku , 这个本地 sku 可以分两种，库存 sku 和组合 sku。其中特殊就在这个组合 sku，这个组合 sku 包含多个库存 sku。
所以汇总的意思就是，备货单只会得到各个库存 sku 具体有多少个的数据。

但是，在发票模板中，我们必须要按照装箱数据分类好这些库存 sku
幸运的是 装箱数据 中有一列是`装箱数量`，根据这个`装箱数量`我们就可以倒推出一切东西

但只有 装箱数据还不够，我们必须知道 MSKU -> 组合SKU -> 对应的一些库存sku，万幸，这个数据可以从发货单中的这两列 `MSKU	` 和 `SKU发货量` 找到

---

不幸的是，备货单中的库存 sku 不会完全对应，实际的库存 sku ，你问为什么？
财务人员，会对`规则型号`和`单价`两个都一样的库存 sku 进行合并。
`规则型号`和`单价`在哪看？备货单的第二个 sheet，在这个 sheet 的第一行可以找到下面的字段：
```xlsx
SKU	产品名称	发货量	规则型号	单价	供货商	余	单价		规则型号	发货量	单价	供货商	采购总价		供货商	深圳市卓越灏科技有限公司	
```
按顺序看，一般合并后剩下的库存 sku 都是`规则型号`和`单价`相同的库存 sku 的第一个。

最终的目的是保证每箱的数量和之前的一致对吧。
他们以`规则型号`进行合并，说明最后的结果就是，一种`规则型号`每箱只有一个 sku 有。
也就是说我这也对每箱的相关的`规则型号`数量进行提前统计，然后合并把这个数量全部放到该箱的该`规则型号`的 sku 上。
