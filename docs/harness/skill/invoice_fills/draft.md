那么，根据什么来填写呢？
还记得填写报关资料用的那个备货单吗，就是那个。
而且填写这个发票模板和填写报关资料的 `申报要素` 非常相似
---
关于备货单的介绍：
    用户会发送一个 excel 文件，
    （命名类似 "4.26-SP260414001-新棱镜备货-美国（4.28）-2.xlsx"，格式大概为“日期-SP单号-其它信息-国家（又是一个日期）”，注意，重要的是中间的单号和国家，在 SKILL 中要求 AI 自己从文件名中提取这两个信息，如果拿不到就和用户说缺少关键信息）
    然后要求执行填写报关单流程。

    这个 xlsx 文件中需要的数据在第三个 sheet。
    这第三个 sheet 第一行一定是下面这样的
    ```xlsx
    日期	SKU	品名	规格型号	发货量	单价	供货商	采购订单号	采购总价	商品名称	售价	总价	单位
    ```
    具体需要脚本提取的列有这些：
    ```xlsx
    品名    规格型号    发货量  商品名称    售价    总价    单位
    ```
    不过具体提取几行还是取决于第一列有几行有效数据。

OK来看具体的填写方式
```xlsx
产品英文品名*	产品中文品名*
```
这两个填写备货单的`商品名称`，英文名称直接做成规则映射，比如：
表带 -> Watch Band
编织表带 -> Braided Watch Band
表壳 -> Watch Case
包装盒 -> Packaging Box
手表保护套 -> Watch Protective Case
---
```xlsx
单箱产品申报数量*	单位*
```
这两个分别对应备货单的`发货量`和`单位`

```xlsx
产品材质*	产品海关编码*
```
这两个分别的填写方式直接复制填写报关资料时填写`其他申报要素`和`海关HS编码`的逻辑代码
但是`产品材质`要加点料，也就是不上英文，格式是这样的`en/cn`，比如
`尼龙/nylon`，`纸质+塑料/paper+plastic`

```xlsx
产品申报单价*
```
这个的填写方式依赖于 `单箱产品申报数量*` 和 `产品材质*`，是这样计算的：表带 + 硅胶 0.35/个
表带 + 尼龙  0.5/个
表带 + 金属（或贱金属）  1/个
表壳（不看材质）  0.35/个
包装盒（不看材质） 0.32/个
手表保护套（不看材质） 0.35/个
其它 如果以上材质都不符合，直接填入`没有该材质的计算价格方式`

产品申报单价填写的是计算好得价格，比如定义为硅胶，有 200 个，就是 0.35 * 200。

```xlsx
产品品牌*	品牌类型*
```
这两个填写很简单，填`无`即可

```xlsx
产品型号*
```
这个的来源是备货单的 `SKU` 这一列

```xlsx
产品图片*
```
这个就有一点绕，我们必须要用备货单的 `SKU` 这一列去下载库存 sku 数据，可以复用现在的发送请求方式，然后提取出 `库存sku图片` 复制过来即可。
图片匹配失败时怎么办：
留空并在 JSON notice 里提示


```xlsx
产品用途*
```
根据以下对应关系填写：
表带	装饰/decorate
表壳	装饰/decorate
手表保护套	保护手表用/Used for protecting watches
包装盒	装表带表壳/Watch strap and case
