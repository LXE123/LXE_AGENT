只能分析出销量的趋势。
不过备货分析好像也只需要销量的趋势。
主要计算如下，
1. 加权日销 = 7天销量 / 7 * 0.6 + 14天销量 / 14 * 0.3 + 30天销量 / 30 * 0.1
2. 销量趋势：
current_speed = 7天销量 / 7
recent_speed = (14天销量 - 7天销量) / 7
older_speed = (30天销量 - 14天销量) / 16

baseline = recent_speed * 0.6 + older_speed * 0.4
trend_ratio = current_speed / baseline

if sales_30d < 10:
    trend = "样本不足"
elif baseline == 0:
    if current_speed > 0:
        trend = "新增出单/恢复出单"
    else:
        trend = "无销量"
elif trend_ratio >= 1.5:
    trend = "快速增长"
elif trend_ratio >= 1.2:
    trend = "增长"
elif trend_ratio <= 0.6:
    trend = "快速下降"
elif trend_ratio <= 0.85:
    trend = "下降"
else:
    trend = "平稳"

但进行这些计算前，我们必须得先做一件事，先格式化数据吧。

这个格式化好的数据应该怎么构成呢？

销量销量，对于亚马逊来说，他既要看的是整个链接的销量，又要看单个 ASIN 的销量。一条链接里会有多个 asin，这些 asin 的销量构成了整个链接的总销量。会出现销量均匀分布到整个链接里的 ASIN，也会出现链接里的某个 ASIN 占据百分之九十的销量。

所以我们格式化，首先就是要按照链接分类好这些 msku。

那么怎么分类呢？答案是`父ASIN`，所有`父ASIN`相同的 msku 都属于同一条链接。

于是我们会进行两种计算，一种是基于链接的计算，一种是基于 asin 的计算。

我们会写一个新的表格，里面有 5 个 sheet，
第一个是销量前 10 的链接的计算结果，
第二个是其余的链接的计算结果
第三个是销量前 50 的 asin 的计算结果
第四个是其余的 asin 的计算结果
第五个是原始的 msku 数据，但额外加上三列，分别是`加权日销`，`销量趋势速率`，`销量趋势`
（我们会计算出`加权日销`，`销量趋势速率`，`销量趋势`）

ok，把上面的流程设计成一个新的 skill，用户会说，我需要xxx店铺的销量分析报告，然后脚本会找到已经下载到本地的 msku 文件，挑选出最新的那一份数据进行操作。记得对比一下日期，如果最新的那一份数据德时间小于当天（提醒只计算到天），提醒用户是否确定就是分析这份数据，还是下载最新的。

我注意到前 4 个 sheet 最后 3 列都是 `加权日销`，`销量趋势速率`，`销量趋势`。我打算在每个 sheet 的 `加权日销` 的前一列都加一列 `商品链接`，ASIN 的 `商品链接` 好加，直接复制就行。
但是链接的 sheet 就需要一些规则，
我们都知道原始 msku 数据中有一列是 `商品链接`，我们在填写链接 sheet 的 `商品链接` 就靠这个。
举个例子：
`美国 http://www.amazon.com/gp/product/B0FJFVPQND`
然后把 `B0FJFVPQND` 去掉，得到 `http://www.amazon.com/gp/product/`，然后填写某个链接的 `商品链接` 时，把该链接的父ASIN 拼接到 `http://www.amazon.com/gp/product/` 上，得到 `http://www.amazon.com/gp/product/<父ASIN>`
