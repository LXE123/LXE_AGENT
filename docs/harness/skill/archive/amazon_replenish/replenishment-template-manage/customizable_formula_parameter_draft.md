这个 skill 会用来自定义公式中的参数。

我可以新建一个模板参数，修改一个模板参数，这些参数存储多份，可以任意切换。

既然有切换那么就必须有命名，这个命名应该有用户来完成，实在不想命名，默认会提供`自定义模板1 2 3 4...`

然后就是修改参数，首先必须看看到底有哪些参数可修改：
如下
```text
加权日销 = 7天销量 / 7 * 0.6 + 14天销量 / 14 * 0.3 + 30天销量 / 30 * 0.1
---
补货天数参数
日销 > 10: 增长90 / 平稳90 / 下降75
日销 > 5:  增长90 / 平稳75 / 下降60
日销 > 2:  增长75 / 平稳60 / 下降45
其他:      增长60 / 平稳45 / 下降45
---
可销售天数 <= 40 -> 空运（急发）
40 < 可销售天数 <= 70 -> 空运
可销售天数 > 70 -> 判断是否海运
---
加权日销 <= 5 不建议海运
加权日销 5-10 -> 90天
加权日销 10-20 -> 95天
加权日销 20-50 -> 100天
加权日销 > 50 -> 100天，不够重量再试110天
总重量 > 60kg 才建议海运
```

参数很多，让用户口头描述修改很不靠谱。所以我的想法是直接把这些参数整理到一个文件里，发送给用户修改，修改好后再发回给 agent，agent 用脚本校验一遍，没问题就保存为模板。
似乎用 xlsx 文件比 JSON 更加直观，好修改
